# Deployment Guide

Three deployment options, in order of recommended simplicity. All preserve the security model: your memory stays under your control.

---

## Option 1 — Fly.io (recommended for cloud)

Fly.io runs Docker containers at the edge. Free tier is sufficient for personal use.

**Security posture:** Your server is on the public internet, protected by JWT auth. Add IP allowlist for extra protection.

### Setup

```bash
# Install Fly CLI
curl -L https://fly.io/install.sh | sh

# Authenticate
fly auth login

# Create the app (first time only)
fly launch --no-deploy
# When prompted:
# - App name: futuro-yourname (must be globally unique)
# - Region: pick closest to you (sjc for Bay Area)
# - Postgres: No
# - Redis: No

# Create a persistent volume for your data
fly volumes create futuro_data --size 1 --region sjc

# Deploy
make deploy
```

### `fly.toml` (generated by `fly launch`, customized)

```toml
app = "futuro-yourname"
primary_region = "sjc"

[build]
  dockerfile = "Dockerfile.prod"

[env]
  PORT = "8080"
  DATA_DIR = "/data"
  MEMORY_DIR = "/data/memory"
  CHROMA_DIR = "/data/chroma"
  DB_PATH = "/data/futuro.db"

[[mounts]]
  source = "futuro_data"
  destination = "/data"

[http_service]
  internal_port = 8080
  force_https = true
  auto_stop_machines = true
  auto_start_machines = true
  min_machines_running = 0

[[services.ports]]
  port = 443
  handlers = ["tls", "http"]
```

### Set secrets

```bash
fly secrets set ANTHROPIC_API_KEY=sk-ant-...
fly secrets set JWT_SECRET=$(openssl rand -hex 32)
fly secrets set USER_PASSWORD_HASH=$(python -c "import bcrypt; print(bcrypt.hashpw(b'your_password', bcrypt.gensalt()).decode())")
fly secrets set CLAUDE_MODEL=claude-sonnet-4-5
fly secrets set GIT_AUTO_COMMIT=true
fly secrets set ALLOWED_ORIGINS=https://futuro-yourname.fly.dev
```

### Deploy

```bash
make deploy
# or:
fly deploy
```

### IP allowlist (optional, recommended)

In Fly.io dashboard → your app → Networking → IP Restrictions. Add your home IP and VPN exit IP.

### Backing up data from Fly.io

```bash
# SSH into the machine
fly ssh console

# Inside the machine:
tar -czf /tmp/futuro-backup.tar.gz /data/memory /data/futuro.db

# Download to local
fly sftp get /tmp/futuro-backup.tar.gz ./backups/
```

## Option 2 — Self-hosted VPS + WireGuard (most secure)

Your server is not exposed to the public internet at all. You access it only through a WireGuard VPN tunnel. No auth bypass possible.

**Security posture:** Maximum. Even without a password, the server is unreachable without the VPN.

**Recommended VPS providers:**
- Hetzner Cloud (CX11: €3.79/month, 2 vCPU, 2GB RAM — sufficient)
- DigitalOcean (Basic Droplet: $4/month)
- Vultr (Cloud Compute: $3.50/month)

### Server setup

```bash
# On the VPS (Ubuntu 24.04):

# Install Docker
curl -fsSL https://get.docker.com | sh
usermod -aG docker ubuntu

# Install WireGuard
apt install -y wireguard

# Clone your repo
git clone https://github.com/your-username/futuro.git /opt/futuro
cd /opt/futuro

# Set environment
cp .env.example .env
nano .env  # Fill in secrets

# Start
docker-compose -f docker-compose.prod.yml up -d
```

### WireGuard setup

On the server:
```bash
# Generate keys
wg genkey | tee /etc/wireguard/server-private.key | wg pubkey > /etc/wireguard/server-public.key
wg genkey | tee /etc/wireguard/client-private.key | wg pubkey > /etc/wireguard/client-public.key

# /etc/wireguard/wg0.conf
[Interface]
PrivateKey = <server-private-key>
Address = 10.0.0.1/24
ListenPort = 51820
PostUp = iptables -A FORWARD -i wg0 -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
PostDown = iptables -D FORWARD -i wg0 -j ACCEPT; iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE

[Peer]
PublicKey = <client-public-key>
AllowedIPs = 10.0.0.2/32

# Start WireGuard
systemctl enable wg-quick@wg0
systemctl start wg-quick@wg0

# Only open port 51820 (WireGuard) in firewall — NOT 80 or 443
ufw allow 51820/udp
ufw enable
```

On your laptop:
```bash
# /etc/wireguard/futuro.conf
[Interface]
PrivateKey = <client-private-key>
Address = 10.0.0.2/24

[Peer]
PublicKey = <server-public-key>
Endpoint = <server-public-ip>:51820
AllowedIPs = 10.0.0.1/32  # Only route server traffic through VPN
PersistentKeepalive = 25

# Connect:
wg-quick up futuro

# Access Futuro:
# http://10.0.0.1:3000  (only reachable when VPN is up)
```

### docker-compose.prod.yml

```yaml
version: '3.8'

services:
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile.prod
    restart: unless-stopped
    env_file: .env
    volumes:
      - futuro_data:/app/data
    expose:
      - "8000"

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile.prod
    restart: unless-stopped
    environment:
      - NEXT_PUBLIC_API_URL=http://backend:8000
    expose:
      - "3000"

  nginx:
    image: nginx:alpine
    restart: unless-stopped
    ports:
      - "80:80"      # Only accessible via WireGuard IP (10.0.0.1)
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
    depends_on:
      - backend
      - frontend

volumes:
  futuro_data:
    driver: local
```

## Option 3 — Local only (laptop)

No cloud, no exposure. Everything stays on your machine.

**Security posture:** Complete. Nothing leaves your laptop except API calls to Anthropic.

**Limitation:** Not accessible from your phone or other devices.

```bash
# Start everything
make dev

# Access at http://localhost:3000
# Runs as long as your laptop is on
```

For auto-start on login (macOS):
```bash
# Create a launchd plist at ~/Library/LaunchAgents/com.futuro.plist
# Or use a simpler approach: add to your shell profile
# alias futuro='cd ~/code/futuro && make dev'
```

## Production Dockerfiles

### `backend/Dockerfile.prod`

```dockerfile
FROM python:3.12-slim

WORKDIR /app

# Install system deps
RUN apt-get update && apt-get install -y git && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY app/ ./app/
COPY alembic/ ./alembic/
COPY alembic.ini .

# Create data directory structure
RUN mkdir -p /app/data/memory /app/data/chroma

# Initialize memory git repo on startup
COPY docker-entrypoint.sh .
RUN chmod +x docker-entrypoint.sh

EXPOSE 8000
ENTRYPOINT ["./docker-entrypoint.sh"]
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### `docker-entrypoint.sh`

```bash
#!/bin/sh
set -e

# Initialize memory git repo if not already initialized
if [ ! -d "/app/data/memory/.git" ]; then
  git config --global user.email "futuro@local"
  git config --global user.name "Futuro"
  git init /app/data/memory
  git -C /app/data/memory commit --allow-empty -m "init"
fi

# Run DB migrations
alembic upgrade head

# Rebuild vector index if chroma dir is empty
if [ -z "$(ls -A /app/data/chroma 2>/dev/null)" ]; then
  python -c "from app.memory.vector_store import StoryVectorStore; import asyncio; asyncio.run(StoryVectorStore().rebuild_index())"
fi

exec "$@"
```

## Makefile deploy targets

```makefile
deploy:                       ## Deploy to Fly.io
	fly deploy

deploy-secrets:               ## Push secrets to Fly.io (run once after setup)
	fly secrets set ANTHROPIC_API_KEY=$(ANTHROPIC_API_KEY)
	fly secrets set JWT_SECRET=$(JWT_SECRET)
	fly secrets set USER_PASSWORD_HASH=$(USER_PASSWORD_HASH)

backup:                       ## Backup data directory with timestamp
	tar -czf backups/futuro-backup-$(shell date +%Y-%m-%d).tar.gz \
	  backend/data/memory backend/data/futuro.db
	@echo "Backup saved to backups/"

restore:                      ## Restore from backup (BACKUP=path/to/backup.tar.gz)
	tar -xzf $(BACKUP) -C .
	make migrate
	make rebuild-index
```
